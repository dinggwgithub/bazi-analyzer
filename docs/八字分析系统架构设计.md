### 2.4 分析引擎 (Engine)

```go
type Engine interface {
    Analyze(input string) (*AnalysisResult, error)
}

type engine struct {
    parser       parser.Parser
    scanner      analyzer.RelationScanner
    reconstructor analyzer.PatternReconstructor
    wangshuai    analyzer.WangshuaiAnalyzer
    xiyong       analyzer.XiyongAnalyzer
    reporter     reporter.Reporter
}

func (e *engine) Analyze(input string) (*AnalysisResult, error) {
    // 1. 解析输入
    bazi, err := e.parser.Parse(input)
    if err != nil {
        return nil, err
    }
    
    // 2. 扫描关系
    relations := e.scanner.Scan(bazi)
    
    // 3. 格局重构
    reconstructed := e.reconstructor.Reconstruct(bazi, relations)
    
    // 4. 旺衰判断
    wangshuai := e.wangshuai.Analyze(reconstructed)
    
    // 5. 喜忌用神
    xiyong := e.xiyong.Analyze(wangshuai.Conclusion, reconstructed)
    
    // 6. 生成报告
    result := e.reporter.Generate(bazi, reconstructed, relations, wangshuai, xiyong)
    
    return result, nil
}
```

## 四、设计原则

### 4.1 单一职责原则
每个模块只负责一个功能：
- Parser 只负责解析
- Scanner 只负责扫描
- Reconstructor 只负责重构
- Wangshuai 只负责旺衰判断
- Xiyong 只负责喜忌推导

### 4.2 开闭原则
- 通过接口定义行为
- 新增分析维度只需实现对应接口
- 规则可配置，不修改核心逻辑

### 4.3 依赖倒置原则
- 高层模块依赖抽象接口
- 具体实现可替换
