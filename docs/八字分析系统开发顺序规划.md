## 第一阶段：基础数据结构和天干地支定义

### 阶段目标
建立系统的数据基础，定义天干、地支、五行等核心数据结构和基础操作。

### 输入
- 八字分析详细总结文档中的基础定义
- 天干地支的五行、阴阳属性表
- 地支藏干定义表

### 输出
- `pkg/core/types.go` - 基础类型定义
- `pkg/core/tiangan.go` - 天干定义与操作
- `pkg/core/dizhi.go` - 地支定义与操作
- `pkg/core/wuxing.go` - 五行定义与操作
- 单元测试文件

### 开发任务

## 第四阶段：格局重构与合化判断

### 阶段目标
实现格局重构功能，判断合化是否成功，进行五行重定和干支重评。

### 输入
- 前三阶段的代码
- 合化成功判断规则（得令、透干、无克）
- 优先级规则（三会 > 三合 > 五合 > 六合）

### 输出
- `pkg/analyzer/reconstructor.go` - 格局重构器
- `pkg/analyzer/hehua.go` - 合化判断逻辑
- 单元测试文件

### 开发任务

#### 4.1 合化结果定义 (hehua.go)
```go
type HeHuaType int

const (
    TianGanWuHeType HeHuaType = iota  // 天干五合

    DeLing      bool          // 是否得令
    TouGan      bool          // 是否透干
    WuZuKe      bool          // 是否无强力阻克

```

type RelationshipAnalyzer struct {

type BenZhuangInfo struct {

    GanZhi     core.GanZhi

type TianGanRoute struct {
