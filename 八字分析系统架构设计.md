# 八字分析系统架构设计

## 一、系统架构总览
本系统采用模块化分层设计，遵循"高内聚、低耦合"原则，便于维护和扩展。

```
┌─────────────────────────────────────────────────┐
│                   CLI 接口层                    │
├─────────────────────────────────────────────────┤
│                   分析引擎层                    │
├───────┬───────┬───────┬───────┬───────┬───────┤
│ 步骤1 │ 步骤2 │ 步骤3 │ 步骤4 │ 步骤5 │ 步骤6 │
│ 扫描  │ 重构  │ 旺衰 │ 喜忌 │ 细节 │ 综合 │
├───────┴───────┴───────┴───────┴───────┴───────┤
│                   数据模型层                    │
├─────────────────┬─────────────────────────────┤
│  基础数据模型   │      分析结果模型           │
├─────────────────┴─────────────────────────────┤
│                   工具函数层                    │
└─────────────────────────────────────────────────┘
```

## 三、核心流程设计

### 八字分析主流程
```go
func AnalyzeBaZi(input string) AnalysisResult {
    // 1. 解析输入，创建命盘
    chart := ParseBaZiInput(input)
    
    // 2. 第一步：扫描全局特殊关系
    step1Result := step1.Scan(chart)
    
    // 3. 第二步：格局重构
    step2Result := step2.Reconstruct(chart, step1Result)
    
    // 4. 第三步：定日主旺衰
    step3Result := step3.Analyze(step2Result.ReconstructedChart)
    
    // 5. 第四步：明喜忌用神
    step4Result := step4.Analyze(step3Result)
    
    // 6. 第五步：细析作用关系
    step5Result := step5.Analyze(step2Result.ReconstructedChart)
    
    // 7. 第六步：综合论断
    step6Result := step6.Synthesize(AllResults{
        Step1: step1Result,
        Step2: step2Result,
        Step3: step3Result,
        Step4: step4Result,
        Step5: step5Result,
    })
    
    return AnalysisResult{
        OriginalChart: chart,
        Step1: step1Result,
        Step2: step2Result,
        Step3: step3Result,
        Step4: step4Result,
        Step5: step5Result,
        Step6: step6Result,
    }
}
```

## 四、数据流向设计
1. 输入字符串 → 解析为 BaZiChart
2. BaZiChart → 扫描器 → 特殊关系列表
3. BaZiChart + 特殊关系 → 重构器 → ReconstructedChart
4. ReconstructedChart → 旺衰分析器 → 旺衰结果
5. 旺衰结果 → 喜忌分析器 → 喜忌结果
6. ReconstructedChart → 细节分析器 → 细节分析结果
7. 所有中间结果 → 综合器 → 最终分析报告

## 五、扩展设计
1. 支持不同流派的算法切换（通过接口实现）
2. 支持插件式扩展新的分析维度
3. 支持多种输出格式（文本、JSON、HTML）
4. 预留岁运分析扩展接口
